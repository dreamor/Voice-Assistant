# 系统架构

## 概述

语音助手是一个端到端的中文语音交互系统，将用户的语音输入转换为文本，经过AI处理后，再将文本响应转换为语音输出。

## 核心组件

### 1. 语音活动检测 (VAD)
- **模块**: `vad.py`
- **功能**: 实时监测麦克风输入，自动检测说话开始和结束
- **技术**: 基于 RMS 能量阈值的声音检测
- **特性**:
  - 自动开始录音（检测到声音）
  - 静默超时自动停止（约1.5秒静默后停止）
  - 最小语音时长过滤

### 2. 语音识别 (ASR)
- **模块**: `cloud_asr.py`
- **功能**: 将音频转换为文本
- **服务**: 阿里云 DashScope Paraformer
- **特性**:
  - 实时识别
  - 支持中英文混合
  - 支持文件或字节输入

### 3. AI 对话
- **模块**: `ai_client.py`, `voice_assistant_ai.py`
- **功能**: 处理用户输入，生成AI回复
- **服务**: OpenRouter API（支持多种模型）
- **特性**:
  - 流式输出
  - 对话上下文保持
  - 支持 Interpreter 模式（代码执行）

### 4. 语音合成 (TTS)
- **模块**: `tts.py`
- **功能**: 将文本转换为语音
- **服务**: Microsoft Edge-TTS
- **特性**:
  - 自然流畅的中文发音
  - 多种音色可选
  - 文本预处理优化

### 5. 音频播放
- **模块**: `audio_player.py`
- **功能**: 播放合成的语音
- **技术**: pygame
- **特性**: 临时文件自动清理

## 工作流程

```
1. 用户按 Enter 开始录音
   │
   ▼
2. VAD 模块监听麦克风
   - 检测到声音 → 开始录音
   - 静默超时 → 停止录音
   │
   ▼
3. 将录制的音频保存为 WAV 格式
   │
   ▼
4. 云端 ASR 识别音频为文本
   │
   ▼
5. AI 处理模块
   - Interpreter 模式: 解析意图，执行代码
   - AI 聊天模式: 生成自然语言回复
   │
   ▼
6. TTS 将文本合成为语音
   │
   ▼
7. 播放器输出到扬声器
   │
   ▼
8. 等待用户下一轮输入
```

## 配置管理

使用 `python-dotenv` 从 `.env` 文件加载配置：

```
.env
├── ASR配置 (ASR_*)
├── LLM配置 (LLM_*)
├── 音频配置 (SAMPLE_RATE, EDGE_TTS_VOICE)
├── VAD配置 (VAD_*)
└── AI配置 (SYSTEM_PROMPT)
```

## 技术栈

| 类别 | 技术 |
|------|------|
| 编程语言 | Python 3.x |
| 语音识别 | 阿里云 DashScope (Paraformer) |
| AI 对话 | OpenRouter API |
| 语音合成 | Microsoft Edge-TTS |
| 音频录制 | sounddevice |
| 音频播放 | pygame |
| 音频处理 | soundfile, numpy |
| 配置管理 | python-dotenv |

## 依赖关系

```
voice_assistant_ai.py (主程序)
    ├── vad.py (录音)
    ├── cloud_asr.py (语音识别)
    ├── ai_client.py (AI对话)
    ├── tts.py (语音合成)
    └── audio_player.py (音频播放)
```

## 扩展点

1. **ASR 引擎**: 可替换为其他语音识别服务
2. **TTS 引擎**: 可替换为其他语音合成服务
3. **AI Provider**: 可通过 OpenRouter 切换不同模型
4. **音频前端**: 可添加降噪、回声消除等预处理
