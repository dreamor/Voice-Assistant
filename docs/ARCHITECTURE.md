# 系统架构

## 概述

语音助手是一个端到端的中文语音交互系统，支持两种工作模式：
- **Interpreter 模式**：检测电脑操作意图，执行代码
- **AI 模式**：纯对话模式，LLM 生成自然语言回复

## 核心组件

### 1. 语音活动检测 (VAD)
- **模块**: `vad.py`
- **功能**: 实时监测麦克风输入，自动检测说话开始和结束
- **技术**: 基于 RMS 能量阈值的声音检测

### 2. 语音识别 (ASR)
- **模块**: `cloud_asr.py`
- **功能**: 将音频转换为文本
- **服务**: 阿里云 DashScope Paraformer

### 3. 智能指令处理
- **模块**: `voice_assistant_ai.py`
- **核心逻辑**: `handle_with_interpreter()`
  - 检测操作关键词（打开/关闭/创建/删除等）
  - 根据意图调用 LLM 生成代码
  - 执行 Python/Bash 代码并返回结果

### 4. AI 对话
- **模块**: `ai_client.py`
- **功能**: 流式 LLM 对话（AI 模式使用）

### 5. 语音合成 (TTS)
- **模块**: `tts.py`
- **服务**: Microsoft Edge-TTS

### 6. 音频播放
- **模块**: `audio_player.py`
- **技术**: pygame

## 工作流程

```
1. 用户按 Enter 开始录音
   │
   ▼
2. VAD 模块监听麦克风
   - 检测到声音 → 开始录音
   - 静默超时 → 停止录音
   │
   ▼
3. 将录制的音频保存为 WAV 格式
   │
   ▼
4. 云端 ASR 识别音频为文本
   │
   ▼
5. 判断指令类型
   │
   ├── Interpreter 模式（默认）
   │   ├── 检测操作关键词
   │   ├── 调用 LLM 生成代码
   │   ├── 执行代码
   │   └── 返回执行结果
   │
   └── AI 模式
       ├── 调用 LLM 流式对话
       └── 返回自然语言回复
   │
   ▼
6. TTS 将文本合成为语音
   │
   ▼
7. 播放器输出到扬声器
   │
   ▼
8. 等待用户下一轮输入
```

## 模式说明

### Interpreter 模式

**触发条件**: 用户指令包含电脑操作关键词

**操作关键词**:
- 文件操作：打开、关闭、创建、删除、移动、重命名、复制
- 系统操作：打开应用、关闭窗口、启动、停止、安装、卸载
- 其他：截屏、截图、搜索、查找、下载、上传、控制、操作

**处理流程**:
1. LLM 解析用户意图
2. 生成可执行 Python/Bash 代码
3. 执行代码并捕获输出
4. 返回执行结果

### AI 模式

**触发条件**:
- 按 `I` 键手动切换
- 非操作指令

**处理流程**:
1. 调用 LLM 生成回复
2. 支持多轮对话上下文
3. 流式输出减少等待时间

## 配置管理

使用 `python-dotenv` 从 `.env` 文件加载配置：

```
.env
├── ASR配置 (ASR_*)
├── LLM配置 (LLM_*)
├── 音频配置 (SAMPLE_RATE, EDGE_TTS_VOICE)
├── VAD配置 (VAD_*)
└── AI配置 (SYSTEM_PROMPT)
```

## 技术栈

| 类别 | 技术 |
|------|------|
| 编程语言 | Python 3.x |
| 语音识别 | 阿里云 DashScope (Paraformer) |
| LLM | 阿里云百炼 (kimi-k2.5) |
| 语音合成 | Microsoft Edge-TTS |
| 音频录制 | sounddevice |
| 音频播放 | pygame |
| 音频处理 | soundfile, numpy |
| 配置管理 | python-dotenv |

## 扩展点

1. **ASR 引擎**: 可替换为其他语音识别服务
2. **TTS 引擎**: 可替换为其他语音合成服务
3. **LLM**: 可通过配置切换不同模型
4. **操作关键词**: 可在代码中自定义扩展
